1.5.3.3.2.1.3.3 风险结论法规依据追溯

本模块用于实现对系统生成的风险结论进行全链路溯源，支持从最终风险判断回溯至原始数据来源、计算过程、规则匹配及法规依据。通过可视化路径展示和结构化信息呈现，帮助用户理解风险判定的逻辑链条，提升决策透明度与合规可解释性。核心功能包括

（1）风险结论列表展示

以卡片式布局展示所有已生成的风险结论，每条记录包含：

风险标题（如“企业存在高流动性风险”）

风险发生时间

当前状态标识（如红色警告三角）

可点击的【查看详情】按钮，进入完整追溯视图

（2）推理路径可视化

每条风险项上方展示其完整的推理路径，以层级标签形式呈现从数据源到最终结论的全过程，例如：

数据采集→指标计算→规则匹配→风险识别→法规映射

用户可通过该路径快速定位关键环节。

（3）数据来源明确标注

清晰列出支撑该风险判断的关键数据项及其来源，如“资产负债表-流动资产”“利润表-净利润”，并标明数据提取方式（如系统接口、人工录入），确保数据可验证。

（4）计算过程透明化展示

提供详细的计算公式与中间结果，例如：

“流动比率 = 流动资产 / 流动负债 = 1,200,000 / 1,600,000 = 0.75”

“连续亏损期 = Q1-Q3累计亏损额>0，持续时间超过12个月”

支持用户逐级查看各阶段的运算逻辑，增强结果可信度。

（5）法规依据精准关联

在“法规依据”区域引用对应的监管文件及具体条款，如《商业银行流动性风险管理办法》第十五条，并附上原文摘要，说明该条款如何支撑当前风险判断。

（6）支持全文搜索与快速定位

顶部提供搜索框，支持按风险结论关键词、法规名称或条款编号进行检索，快速定位目标风险项，提升操作效率。

（7）多维度风险联动展示

当一个企业存在多个风险点时，系统可将相关联的风险结论集中展示，体现其内在逻辑关系（如“流动性不足→偿债能力下降→信用风险上升”），支持综合研判。
