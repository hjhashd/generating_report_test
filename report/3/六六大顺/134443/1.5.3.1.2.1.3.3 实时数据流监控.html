1.5.3.1.2.1.3.3 实时数据流监控

“实时数据流监控”支持对Kafka、REST API、数据库主库、S3等多类数据源进行秒级监控。通过持续采集日志与元数据，以仪表盘形式实时展示吞吐量、连接状态、解析成功率、Schema变更等关键指标，并结合数据预览和事件流，自动识别如字段类型变更、新增字段、格式识别失败、元数据提取成功等典型事件，实现异常的即时发现、精准定位与快速响应。在南网财务公司等高并发、多源异构场景中，该功能有效保障数据接入质量，降低业务中断风险，支撑信贷风控、审计、报表等关键业务，推动数据治理从被动处置转向主动防御，夯实智能数据中台基础。核心功能如下：

（1）数据吞吐量

显示当前单位时间内的数据摄入总量，并标注环比增长率，反映数据流入趋势。

（2）活跃连接数

展示当前正在正常通信的数据源数量，标识系统连接稳定性。

（3）格式解析成功率

统计数据在接入过程中成功解析的比例，若低于阈值将触发告警。

（4）Schema 漂移待处理数

显示当前待处理的 Schema 变更事件数量点击可跳转至“Schema 管理”页面进行处置。

所有指标均支持实时刷新，支持自定义时间范围筛选。

（5）实时事件流事件类型丰富

自动捕获并分类展示各类数据处理事件，包括：

Schema 更新检测：如user_email类型由string→varchar(255)；

新字段发现：如REST API 中新增created_at字段；

格式自动识别：如成功识别Parquet格式并展示解析的记录；

Schema 兼容处理：如age字段从int扩展为bigint已自动适配；

元数据提取：如从PNG图像中成功提取EXIF元数据；

解析失败告警：如JSON格式校验失败，提示具体错误位置。

（6）时间轴展示

按时间顺序排列事件，精确到秒（如14:32:15），便于追溯问题发生时间点；

（7）操作支持：

支持点击事件查看详细信息、跳转至对应处理页面（如Schema管理、映射配置）；

（8）事件标签化

使用颜色区分事件级别（蓝色=正常，红色=告警，黄色=警告），提升可读性。

（9）数据预览

实时样本展示提供最近数据流样本的结构化预览，直观展示实际数据内容与格式。

（10）格式识别

自动标注每条记录的原始格式（JSON、Avro、Parquet等），辅助判断解析准确性；

（11）操作支持

“查看全部”：跳转至完整数据浏览页；

“导出”：将样本数据导出为CSV或JSON文件，用于调试或审计。
