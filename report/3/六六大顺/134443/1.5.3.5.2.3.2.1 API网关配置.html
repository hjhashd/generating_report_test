1.5.3.5.2.3.2.1 API网关配置

网关配置功能通过集中化管理API接口的访问策略与流量控制参数，实现对各类业务接口的统一接入控制与安全防护。该功能支持对接口进行命名、路径定义、方法设置、限流配置及状态管理，确保系统在高并发场景下稳定运行，并保障接口调用的安全性与可控性。核心功能包括：

（1）API接口列表展示

以表格形式列出当前已配置的所有API接口，每项包含以下字段：

API名称：如“信用评分接口”“风险评估接口”“客户查询接口”；

路径：标识接口的访问地址，如/api/v1/credit/score；

方法：定义请求方式，如 POST 或 GET；

限流配置：显示接口允许的最大调用频率，如“1000次/分钟”；

状态：标明接口当前运行状态，如“启用”或“停用”；

操作按钮：提供“配置”和“停用”选项，支持动态调整接口行为。

（2）接口限流策略管理

系统支持为每个API设置独立的调用频率限制，防止单一接口因异常请求导致服务过载。例如：

信用评分接口设为1000次/分钟；

风险评估接口设为500次/分钟；

客户查询接口设为2000次/分钟。

通过差异化限流规则，平衡系统负载与业务需求。

（3）接口状态控制与启用/停用操作

管理员可通过“启用”或“停用”按钮快速切换接口的对外服务状态，实现临时维护、故障隔离或版本切换等场景下的灵活管控，无需修改代码即可完成服务上下线。

（4）新增与配置入口支持
页面右上角提供“新增API”按钮，支持添加新的接口配置项。点击后可进入配置界面，填写API名称、路径、请求方法、限流规则等信息，完成接口接入流程，实现网关资源的动态扩展与统一管理。
