1.8.3.3.3.3 里程碑复审

在每一个阶段结束之前执行里程碑复审，其目的是要复审和评估本里程碑的目标是否达到。通常在进行里程碑目标评估前，先要对本阶段进行验收复审，复审是否按计划完成了本阶段的所要做的事情，哪些事情已经完成了。在复审和评估里程碑目标的时候，根据目前完成和还没有完成的工作范围，以及本阶段的风险规避的情况，去评估里程碑的目标是否达到。

项目阶段

里程碑目标

启动阶段

1、项目当前开发周期的范围和目标的到确定。从涉众角度，获得对需求范围的一致认识，从而获得对需求范围的稳定性，即在需求范围上的风险已经很大程度得到规避。
2、基于相对稳定的需求范围，整个开发周期的总体工作量和进度得到估算。
3、项目潜在的风险得到评估。

深化阶段

1、构架、需求和计划足够的稳定性得到确保。
2、构架上重要的风险得到处理
3、需求从用户和技术角度的重大风险都得到解决，从用户的角度，使得直接影响用户的需求得到公司的理解和认同，减少因用户因素引起的需求变更，从技术角度，使得不会因为技术不能实现从而变更需求。

构建阶段

1、所有需求的分析、开发和研发内部测试完成。
2、开发效率和质量达到最大化。
3、确保已为部署系统做好准备。

产品化阶段

1、当前开发版本的软件系统成功部署到客户环境。
2、当前开发版本的软件系统完成用户验收。
