# Git 使用与备份指南 - generate_report_test

本文档旨在介绍 `generate_report_test` 项目的 Git 配置情况，并指导如何进行后续的代码备份与推送。

## 1. 项目概况
本目录是一个独立的报告生成系统模块，主要功能包括：
- **AI 摘要生成**：利用 AI 技术对内容进行智能总结。
- **报告自动化**：自动化生成和处理报告文档。
- **容器化支持**：包含 Docker 配置，支持快速部署。

## 2. 当前 Git 配置
为了确保不干扰原始项目的设置，我们采用了“备份仓库”模式：

- **当前分支**：`main`
- **远程备份名**：`backup`
- **远程地址**：`git@github.com:hjhashd/generating_report_test.git` (SSH 方式)

> **注意**：我们将远程仓库命名为 `backup` 而非默认的 `origin`，是为了将其标记为备份用途。

## 3. 下次推送指南
当您对代码进行了修改，想要推送到备份仓库时，请按照以下步骤操作：

### 步骤 A：查看更改状态
在推送前，建议先确认修改了哪些文件：
```bash
git status
```

### 步骤 B：提交更改
1. **添加文件**：将修改后的文件添加到暂存区。
   ```bash
   git add .
   ```
2. **记录说明**：编写本次修改的备注信息。
   ```bash
   git commit -m "您的修改说明 (例如：修复了XX功能)"
   ```

### 步骤 C：推送到备份仓库
由于远程仓库名为 `backup`，推送命令如下：
```bash
git push backup main
```

## 4. 忽略规则 (.gitignore)
为了保持仓库整洁，以下内容已被配置为自动忽略，不会被提交：
- `pycache/` (Python 编译缓存)
- `report/` (生成的临时报告)
- `.env` (本地环境变量，包含敏感信息)
- `editor_image/` (编辑器缓存图片)

## 5. 常用命令速查
- **查看远程配置**：`git remote -v`
- **查看提交历史**：`git log --oneline`
- **撤销上次提交（保持文件修改）**：`git reset --soft HEAD~1`
